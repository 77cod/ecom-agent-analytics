## 1.1 5 分钟跑起来

> 🔧 **实战**：跟着以下步骤，最快 5 分钟看到项目运行效果。

### 第一步：启动基础设施

```bash
# 在项目根目录执行
$ docker compose up -d
```

这个命令会在后台启动 6 个服务：PostgreSQL、Redis、Milvus、Elasticsearch、MinIO、etcd。

验证一下是否都启动了：

```bash
$ docker compose ps
```

你应该看到 6 个服务状态都是 `Up`。

### 第二步：配置 API Key

在后端目录创建 `.env` 文件：

```bash
$ cd backend
$ echo 'DASHSCOPE_API_KEY=你的阿里百炼key' > .env
$ echo 'BOCHA_API_KEY=你的博查搜索key' >> .env
```

> 💡 **提示**：阿里百炼 API Key 在 [bailian.console.aliyun.com](https://bailian.console.aliyun.com) 免费申请。博查搜索 Key 在 [open.bochaai.com](https://open.bochaai.com) 申请。两个都有免费额度。

### 第三步：启动后端

```bash
$ cd backend
$ pip install -r requirements.txt
$ python -m uvicorn app.app_main:app --reload --port 8000
```

看到 `Application startup complete` 说明后端启动成功。

打开浏览器访问 `http://localhost:8000/docs`，你会看到 FastAPI 自动生成的 Swagger API 文档，列出了所有接口。

### 第四步：启动前端

```bash
$ cd frontend
$ npm install
$ npm run dev
```

打开浏览器访问 `http://localhost:5173`，你会看到登录页面。

### 第五步：注册并登录

点击"注册"，输入用户名和密码，然后登录。现在你可以看到主页了！

左侧是行业选择器和导航栏，右侧是内容区。

## 1.2 项目是什么——一句话讲清楚

这是一个**智能电商运营分析平台**，核心能力是：

```
用户选行业 + 输入问题 → 7个AI Agent自动协作 → 运营分析报告 + 可视化图表 + 运营看板
```

举个例子：你选"美妆个护"行业，输入"分析精华液品类本月销售趋势和竞品动态"，系统会：

1. 自动拆解成几个分析维度（销售趋势、商品排行、评价分析、竞品对比、达人ROI…）
2. 查询模拟的电商经营数据
3. 提取 GMV、好评率、转化率等关键指标
4. 生成折线图、饼图、柱状图
5. 写一份完整的 Markdown 运营分析报告
6. 审核质量，不合格自动修订
7. 在运营看板里展示 KPI + 排行榜 + 图表

|------|------|----------|-----------|

### AI 模型

| 模型 | 使用场景 | 特点 |
|------|----------|------|
| DeepSeek-V3.2 | Architect/DataAnalyst/Wizard/Critic/Writer | 推理能力强，适合复杂分析 |

### 数据流核心路径

1. **用户输入** → 前端发请求到后端
2. **路由层** → 校验参数，分发到对应的 Service
3. **DeepResearchGraph** → 按计划执行 Agent 流水线
4. **Agents** → 读写共享状态 ResearchState，通过 asyncio.Queue 推送 SSE 事件
5. **前端** → 接收 SSE 流，实时更新 UI（研究步骤、图表、报告）

## 1.6 7 个 AI Agent 分工

| Agent | 中文名 | 模型 | 职责 | 输入 | 输出 |
|-------|--------|------|------|------|------|
| ChiefArchitect | 分析架构师 | deepseek-v3.2 | 分析问题，制定分析大纲 | 用户问题 | 5-8个分析维度 |
| DeepScout | 信息侦察员 | qwen-plus | 网络搜索（Bocha API） | 搜索关键词 | 网页内容 |
| DataQueryer | 数据查询员 | qwen-plus | 查询模拟数据库 | 大纲维度 | 结构化事实 |
| DataAnalyst | 数据分析师 | deepseek-v3.2 | 提取关键指标 | 事实数据 | GMV/ROI等指标 |
| CodeWizard | 代码极客 | deepseek-v3.2 | 生成代码+图表 | 数据指标 | ECharts/图片 |
| LeadWriter | 首席写手 | deepseek-v3.2 | 撰写分析报告 | 全文数据 | Markdown报告 |
| CriticMaster | 毒舌评论家 | deepseek-v3.2 | 审核报告质量 | 报告草稿 | 评分+修改建议 |

## 1.7 Agent 流水线

```
用户输入："分析美妆精华液销售趋势"
    │
    ▼
┌──────────────┐    Phase 1: PLAN
│ Architect    │    分析问题 → 生成大纲
│ 分析架构师    │    输出: 6个分析维度, 5个研究假设
└──────┬───────┘
       ▼
┌──────────────┐    Phase 2: RESEARCH
│ DataQueryer  │    查询美妆模拟数据
│ 数据查询员    │    输出: 商品数据, 订单, 评价, 达人数据
└──────┬───────┘
       ▼
┌──────────────┐    Phase 3: ANALYZE
│ DataAnalyst  │    提取指标 (GMV, ROI, 好评率)
│ + CodeWizard │    生成图表 (折线图, 饼图, 柱状图)
└──────┬───────┘
       ▼
┌──────────────┐    Phase 4: WRITE
│ LeadWriter   │    撰写 Markdown 运营报告
│ 首席写手      │    整合数据 + 图表 + 洞察
└──────┬───────┘
       ▼
┌──────────────┐    Phase 5: REVIEW
│ CriticMaster │    评分 < 6.0 → 打回修订
│ 毒舌评论家    │    评分 ≥ 6.0 → 完成
└──────────────┘
```
