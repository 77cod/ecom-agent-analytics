# 常用命令速查手册

> 智能电商运营分析平台项目日常开发中使用的全部命令，按类别整理，每条附带一句话说明。

---

## 一、Docker 命令

### 启动与停止

```bash
# 启动所有服务（后台运行，自动构建镜像）
docker compose up -d --build

# 仅启动指定服务（如只启动后端和数据库）
docker compose up -d backend postgres redis

# 停止所有服务
docker compose down

# 停止所有服务并删除数据卷（谨慎，会清除数据库数据）
docker compose down -v
```

### 状态查看

```bash
# 查看所有服务运行状态
docker compose ps

# 查看所有容器（包括已停止的）
docker compose ps -a

# 查看 CPU / 内存占用
docker stats
```

### 日志

```bash
# 查看所有服务日志（实时滚动）
docker compose logs -f

# 查看指定服务最近 100 行日志
docker compose logs --tail=100 backend

# 查看指定服务日志并跟随
docker compose logs -f backend
```

### 重启与重建

```bash
# 重启指定服务
docker compose restart backend

# 重新构建并启动指定服务（代码改动后）
docker compose up -d --build backend

# 仅重新构建不启动
docker compose build backend
```

### 进入容器调试

```bash
# 进入后端容器 bash
docker compose exec backend bash

# 进入容器并启动 Python 交互环境
docker compose exec backend python

# 进入 PostgreSQL 容器运行 psql
docker compose exec postgres psql -U postgres -d industry_assistant
```

## 二、Python / 后端命令

### 依赖管理

```bash
# 安装项目依赖
pip install -r requirements.txt

# 安装单个包
pip install fastapi

# 导出当前环境依赖到文件
pip freeze > requirements.txt
```

### 启动开发服务器

```bash
# 以开发模式启动 FastAPI（热重载）
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 指定环境变量文件启动
uvicorn app.main:app --reload --env-file .env
```

### 代码检查

```bash
# 使用 black 格式化代码
black app/

# 使用 isort 整理导入顺序
isort app/

# 使用 ruff 检查代码风格
ruff check app/

# 使用 mypy 进行类型检查
mypy app/
```

### 测试与调试

```bash
# 运行所有测试
pytest

# 运行指定测试文件
pytest tests/test_agent.py

# 显示详细输出并进入 pdb 调试
pytest -s --pdb

# 使用 Python 快速测试一段代码
python -c "from app.service.deep_research_v2.agents import PlannerAgent; print('import OK')"
```

### 数据库迁移

```bash
# 生成迁移文件（自动检测模型变更）
alembic revision --autogenerate -m "描述本次变更"

# 执行迁移到最新版本
alembic upgrade head

# 回滚到上一个版本
alembic downgrade -1

# 查看当前版本
alembic current
```

## 三、Node / 前端命令

### 依赖管理

```bash
# 安装项目依赖
npm install

# 安装指定包
npm install axios

# 安装开发依赖
npm install -D @types/react

# 清除缓存（遇到依赖问题时尝试）
npm cache clean --force
```

### 开发与构建

```bash
# 启动开发服务器（Vite 默认端口 5173）
npm run dev

# 生产构建
npm run build

# 预览生产构建结果
npm run preview

# TypeScript 类型检查（不生成文件）
npx tsc --noEmit
```

### 代码检查

```bash
# ESLint 检查
npx eslint src/ --ext .ts,.tsx

# ESLint 自动修复
npx eslint src/ --ext .ts,.tsx --fix

# Prettier 格式化
npx prettier --write "src/**/*.{ts,tsx,scss}"
```

## 四、Git 命令

### 基本操作

```bash
# 克隆仓库
git clone <仓库地址>

# 查看当前状态（哪些文件有改动）
git status

# 将文件加入暂存区
git add <文件名>
git add .                   # 添加所有改动

# 提交
git commit -m "feat: 添加用户登录功能"

# 推送到远程
git push origin main

# 拉取远程更新
git pull origin main
```

### 分支操作

```bash
# 查看所有分支
git branch -a

# 创建并切换到新分支
git checkout -b feature/agent-dashboard

# 切换到已有分支
git checkout main

# 删除本地分支
git branch -d feature/old-branch

# 合并分支（先切换到目标分支）
git checkout main
git merge feature/agent-dashboard
```

### 历史查看

```bash
# 查看提交历史
git log --oneline

# 查看最近 10 条图形化历史
git log --oneline --graph -10

# 查看某个文件的修改历史
git log -p -- <文件路径>
```

### 撤销操作

```bash
# 撤销工作区改动（恢复到最后一次提交状态）
git checkout -- <文件名>

# 撤销暂存区文件（保留工作区改动）
git reset HEAD <文件名>

# 回退最近一次提交（保留改动在工作区）
git reset --soft HEAD~1

# 回退最近一次提交（丢弃改动）
git reset --hard HEAD~1

# 创建反向提交来撤销
git revert HEAD
```

## 五、数据库命令

### PostgreSQL

```bash
# 连接到数据库
psql -U postgres -d industry_assistant

# 列出所有数据库
\l

# 列出所有表
\dt

# 查看表结构
\d 表名

# 执行 SQL 查询
SELECT * FROM users LIMIT 5;

# 退出 psql
\q
```

### Redis

```bash
# 连接 Redis
redis-cli

# 查看所有键
KEYS *

# 查看键类型
TYPE 键名

# 获取字符串键值
GET 键名

# 删除键
DEL 键名

# 清空当前数据库所有键
FLUSHDB

# 退出
exit
```

## 六、API 测试（curl 示例）

### 用户认证

```bash
# 注册新用户
curl -X POST http://localhost:8000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","email":"test@example.com","password":"123456"}'

# 登录获取 JWT Token
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"123456"}'
```

### 分析服务

```bash
# 发起深度研究分析（携带 Token）
curl -X POST http://localhost:8000/api/v1/analysis/research \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-jwt-token>" \
  -d '{"topic":"2024年Q3智能手机市场趋势","depth":"deep"}'

# 查询分析历史
curl -X GET http://localhost:8000/api/v1/analysis/history \
  -H "Authorization: Bearer <your-jwt-token>"
```

### 健康检查

```bash
# 后端健康检查
curl http://localhost:8000/health

# 查看 API 文档（在浏览器中打开）
# http://localhost:8000/docs
```
