# 术语表

> 智能电商运营分析平台项目全部技术术语速查，按模块分组，每项提供英文原名、中文译名和一句话解释。

## 前端（Frontend）

| 术语 | 解释 |
|------|------|
| **React（React）** | Facebook 开源的前端 UI 框架，采用组件化方式构建用户界面，本项目使用 React 18。 |
| **TypeScript（TypeScript）** | JavaScript 的超集，增加了静态类型系统，让代码在编写阶段就能发现类型错误。 |
| **Vite（Vite）** | 新一代前端构建工具，利用浏览器原生 ES Module 实现极速冷启动和热更新。 |
| **Ant Design（Ant Design）** | 蚂蚁集团开源的企业级 UI 组件库，提供表格、表单、布局等开箱即用的 React 组件。 |
| **ECharts（ECharts）** | Apache 开源的数据可视化图表库，本项目用于渲染运营分析看板中的折线图、柱状图、饼图等。 |
| **Valtio（Valtio）** | 基于 Proxy 的轻量级 React 状态管理库，无需 Provider 包裹即可实现组件间状态共享。 |
| **React Router（React Router）** | React 官方推荐的路由库，管理前端页面的 URL 跳转和导航，本项目使用 v6 版本。 |
| **Axios（Axios）** | 基于 Promise 的 HTTP 客户端，用于向 FastAPI 后端发送请求并处理响应。 |
| **SSE（Server-Sent Events / 服务器推送事件）** | 一种 HTTP 长连接技术，允许后端向前端单向推送数据流，本项目用于 Agent 分析过程的实时展示。 |
| **ES Module（ES 模块）** | ECMAScript 官方模块化标准，使用 `import/export` 语法组织代码，替代传统的 CommonJS。 |
| **JSX（JSX）** | JavaScript 的语法扩展，允许在 JS 代码中直接书写类似 HTML 的标签，React 组件的标准写法。 |
| **TSX（TSX）** | TypeScript 版本的 JSX，即在 TS 文件中使用 JSX 语法，本项目中所有组件文件均以 `.tsx` 结尾。 |
| **Proxy（代理对象）** | ES6 引入的特性，可以拦截对目标对象的读写操作，Valtio 依赖 Proxy 实现自动状态追踪。 |
| **Hook（Hook）** | React 16.8 引入的函数式 API，让函数组件也能使用状态和生命周期等功能，以 `use` 开头命名。 |
| **useEffect（useEffect）** | React Hook，用于处理副作用操作，如数据请求、DOM 操作、定时器等，在组件渲染后执行。 |
| **useState（useState）** | React Hook，为函数组件添加局部状态变量，返回当前值和更新函数。 |
| **useRef（useRef）** | React Hook，创建一个可变的引用对象，常用于保存 DOM 节点引用或跨渲染周期保留值。 |
| **useMemo（useMemo）** | React Hook，缓存计算结果，仅在依赖项变化时重新计算，避免不必要的重复运算。 |
| **useCallback（useCallback）** | React Hook，缓存函数引用，避免因父组件重渲染导致子组件不必要的重新渲染。 |
| **useSnapshot（useSnapshot）** | Valtio 提供的 Hook，将 Proxy 代理状态转为不可变快照，组件仅在所用到字段变化时重渲染。 |
| **Vite HMR（Vite 热模块替换）** | Vite 的热更新机制，修改代码后浏览器无需整页刷新，仅替换变化模块，保持页面状态不丢失。 |
| **CSS Modules（CSS 模块）** | 一种 CSS 作用域隔离方案，为每个类名自动添加哈希后缀，避免样式全局污染。 |
| **SCSS（SCSS / Sassy CSS）** | CSS 预处理器的一种语法，支持变量、嵌套、混入等高级特性，编译后生成标准 CSS。 |

## 后端（Backend）

| 术语 | 解释 |
|------|------|
| **FastAPI（FastAPI）** | 现代高性能 Python Web 框架，基于 Starlette 和 Pydantic，原生支持异步和自动生成 OpenAPI 文档。 |
| **Uvicorn（Uvicorn）** | 基于 uvloop 和 httptools 的 ASGI 服务器，用于启动和运行 FastAPI 应用。 |
| **ASGI（Asynchronous Server Gateway Interface / 异步服务器网关接口）** | Python Web 服务器与应用之间的异步通信标准，是 WSGI 的异步继任者。 |
| **LangGraph（LangGraph）** | LangChain 团队推出的有状态 Agent 编排框架，通过图结构定义多步骤 Agent 工作流并自动管理中间状态。 |
| **SQLAlchemy（SQLAlchemy）** | Python 最流行的 ORM 框架，将数据库表映射为 Python 类，支持使用 Python 代码操作数据库。 |
| **Pydantic（Pydantic）** | Python 数据校验库，利用类型注解自动校验和序列化数据，FastAPI 的请求/响应模型基于此构建。 |
| **TypedDict（类型化字典）** | Python 3.8+ 的类型注解功能，用于精确描述字典中每个键的类型，本项目用于定义 LangGraph 的 Agent 状态结构。 |
| **async/await（异步/等待）** | Python 异步编程核心语法，`async` 定义协程函数，`await` 挂起当前协程等待异步操作完成。 |
| **asyncio（asyncio）** | Python 标准库中的异步 I/O 框架，提供事件循环、协程、任务等基础设施。 |
| **middleware（中间件）** | 介于请求和最终处理函数之间的逻辑层，用于统一处理认证、日志、CORS 等横切关注点。 |
| **CORS（Cross-Origin Resource Sharing / 跨域资源共享）** | 浏览器安全机制，控制不同源的 Web 页面能否访问后端 API，FastAPI 通过 CORSMiddleware 配置允许的前端域名。 |
| **JWT（JSON Web Token / JSON Web 令牌）** | 一种紧凑的令牌格式，用于前端与后端之间的无状态身份认证，包含签名防篡改。 |
| **bcrypt（bcrypt）** | 一种密码哈希算法，将用户密码加密为不可逆的哈希值存储，防止明文泄露。 |
| **dependency injection（依赖注入）** | 一种设计模式，由框架自动创建并注入所需依赖，FastAPI 通过 `Depends()` 函数实现，本项目用于注入数据库会话和当前用户。 |
| **lifespan（生命周期）** | FastAPI 的应用生命周期管理机制，在应用启动和关闭时执行资源初始化和清理操作（如连接数据库、加载模型）。 |

## 数据库与存储（Database & Storage）

| 术语 | 解释 |
|------|------|
| **PostgreSQL（PostgreSQL）** | 功能强大的开源关系型数据库，本项目用作主存储，存放用户、报告、配置等结构化业务数据。 |
| **Redis（Redis）** | 开源内存数据结构存储，用作缓存层和会话管理，大幅提升高频读取数据的响应速度。 |
| **Milvus（Milvus）** | 开源向量数据库，专门存储和检索高维向量数据，本项目用于商品描述、用户评论等的语义相似度搜索。 |
| **Elasticsearch（Elasticsearch）** | 分布式搜索引擎，基于 Lucene 构建，提供强大的全文检索和聚合分析能力，用于商品和订单的复杂搜索。 |
| **MinIO（MinIO）** | 兼容 Amazon S3 API 的对象存储服务，本项目用于存储用户上传的商品图片、报表截图等非结构化文件。 |
| **etcd（etcd）** | 分布式键值存储系统，用作服务注册发现和配置管理，Milvus 集群依赖 etcd 协调各节点状态。 |
| **ORM（Object-Relational Mapping / 对象关系映射）** | 一种用面向对象的方式操作数据库的技术，用 Python 类代表数据库表，用属性代表字段。 |
| **vector database（向量数据库）** | 专为人工智能场景设计的数据库，将文本/图片转为高维数学向量后存储，支持近邻检索（"找出最相似的 N 个"）。 |
| **embedding（嵌入向量）** | 将文字、图片等信息通过模型映射为固定长度的浮点数向量，语义相近的内容向量距离也相近。 |
| **semantic search（语义搜索）** | 基于语义理解的搜索方式，而非关键词机械匹配，能识别"手机壳"和"手机保护套"是同一类商品。 |
| **full-text search（全文搜索）** | 对文本内容建立倒排索引后实现的关键词搜索，支持分词、相关性排序和高亮显示。 |

## AI 与 Agent（AI & Agent）

| 术语 | 解释 |
|------|------|
| **LLM（Large Language Model / 大语言模型）** | 通过海量文本训练出的超大规模神经网络模型，如 GPT、DeepSeek 等，具备理解和生成自然语言的能力。 |
| **Agent（智能体）** | 在 LLM 之上封装了工具调用、记忆和多步推理能力的自主程序，能根据目标自动规划和执行任务。 |
| **Multi-Agent（多智能体）** | 多个 Agent 协作完成复杂任务的架构，本项目将运营分析拆解为规划、搜索、分析等不同角色的 Agent 协同工作。 |
| **prompt（提示词）** | 发送给 LLM 的输入文本，通过精心设计的提示词引导模型输出符合预期的内容。 |
| **system_prompt（系统提示词）** | 在对话开始时设置的全局指令，定义 Agent 的角色、行为准则和输出格式，贯穿整个对话过程直至被覆盖。 |
| **temperature（温度参数）** | LLM 生成文本时的随机性控制参数，取值范围 0~2，值越低输出越确定和保守，值越高输出越有创造性。 |
| **max_tokens（最大令牌数）** | 限制 LLM 单次输出最大长度的参数，防止消耗过多 Token 配额或响应时间过长。 |
| **SSE streaming（SSE 流式传输）** | 通过 SSE 协议将 LLM 的生成结果逐字推送到前端，让用户实时看到分析内容，而非等待全部完成才显示。 |
| **checkpoint（检查点）** | LangGraph 中保存 Agent 工作流中间状态的快照，支持暂停、恢复和回溯到之前任意步骤。 |
| **LangGraph StateGraph（LangGraph 状态图）** | LangGraph 核心类，通过节点（Node）和边（Edge）定义 Agent 工作流的步骤和流转逻辑，每个节点读写共享的 TypedDict 状态。 |
| **TypedDict state（TypedDict 状态）** | 用 Python TypedDict 定义的结构化状态对象，包含 Agent 工作流中的消息历史、中间结果和控制标志等所有上下文信息。 |
| **token（令牌）** | LLM 处理文本的最小单位，一个中文字约等于 1~2 个 token，API 按 token 消耗量计费并限制上下文窗口大小。 |

## DevOps

| 术语 | 解释 |
|------|------|
| **Docker（Docker）** | 容器化平台，将应用及其依赖打包为轻量级容器，确保开发、测试和生产环境完全一致。 |
| **Docker Compose（Docker Compose）** | Docker 官方编排工具，通过 YAML 文件一键启动和管理多容器应用（后端 + 前端 + 数据库 + Redis + Milvus 等）。 |
| **container（容器）** | Docker 中的运行实例，类似于轻量级虚拟机，拥有独立的文件系统、网络和进程空间。 |
| **image（镜像）** | 容器的静态模板，包含应用代码、运行时、依赖库等所有运行所需内容，容器从镜像启动。 |
| **volume（数据卷）** | Docker 的持久化存储机制，将容器内的数据目录映射到宿主机，容器删除后数据不会丢失。 |
| **port mapping（端口映射）** | 将宿主机端口与容器内部端口绑定，使外部能通过 `localhost:宿主机端口` 访问容器内的服务。 |
| **environment variable（环境变量）** | 在操作系统级别定义的键值对配置，用于向程序传递数据库地址、密钥等运行参数，避免硬编码。 |
| **.env（环境变量文件）** | 存放在项目根目录的文本文件，以键值对格式记录环境变量，Docker Compose 和部分框架会自动加载。 |
| **git（Git）** | 分布式版本控制系统，跟踪代码变更历史，支持多人协作、分支管理和版本回溯。 |
| **.gitignore（Git 忽略文件）** | Git 配置文件，列出不需要纳入版本管理的文件和目录（如 `node_modules`、`.env`、`__pycache__` 等）。 |
