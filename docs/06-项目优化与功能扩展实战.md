**常见 strict 报错及修复：**

| 错误 | 修复方式 |
|------|---------|
| `'xxx' is possibly 'undefined'` | 使用可选链 `?.` 或非空断言 `!` |
| `Parameter 'x' implicitly has 'any' type` | 显式声明类型 `x: string` |
| `Property 'x' does not exist on type '{}'` | 定义 interface 类型 |
| `Not all code paths return a value` | 确保函数所有分支都有 return |

**示例修复：**

```typescript
// 修复前
function getIndustryName(id: string) {
  const industry = INDUSTRY_CONFIGS.find(i => i.id === id)
  return industry.name  // TS 报错: Object is possibly 'undefined'
}

// 修复后
function getIndustryName(id: string): string {
  const industry = INDUSTRY_CONFIGS.find(i => i.id === id)
  return industry?.name ?? '未知行业'  // 使用可选链和默认值
}
```

### 6.2.3 API 响应缓存 with Redis

**当前项目的 Redis 使用（`redis_client.py`）：**

```python
# D:/industry_information_assistant/backend/app/core/redis_client.py
class RedisCache:
    def get(self, key: str) -> Optional[Any]:
        try:
            value = self.client.get(key)
            if value:
                return json.loads(value)
            return None
        except Exception as e:
            print(f"Redis get error: {e}")
            return None

    def set(self, key: str, value: Any, expire: int = 3600) -> bool:
        try:
            self.client.setex(key, expire, json.dumps(value, ensure_ascii=False))
            return True
        except Exception as e:
            print(f"Redis set error: {e}")
            return False
```

**可以为研究结果添加缓存：**

```python
# 在 research_router.py 中添加缓存层
from backend.app.core.redis_client import cache
import hashlib

def get_cache_key(query: str, industry_id: str) -> str:
    """生成研究结果的缓存 key"""
    content = f"{query}:{industry_id}"
    hash_val = hashlib.md5(content.encode()).hexdigest()
    return f"research_result:{hash_val}"

@router.post("/stream", status_code=HTTP_200_OK)
async def stream_research(request: ResearchRequest, ...):
    # 检查缓存（同 query + industry 的请求可以复用结果）
    cache_key = get_cache_key(request.query, request.industry_id)
    cached = cache.get(cache_key)
    if cached:
        # 直接返回缓存的结果
        ...

    # 执行研究...

    # 研究完成后缓存结果（过期时间 1 小时）
    cache.set(cache_key, result, expire=3600)
```

**缓存策略总结：**

| 数据类型 | 缓存时间 | 缓存 Key 模式 |
|---------|---------|-------------|
| 行业配置列表 | 24 小时 | `industry_configs` |
| 研究结果 | 1 小时 | `research_result:{md5(query+industry)}` |
| 热门关键词趋势 | 6 小时 | `trend_keywords:{industry_id}` |
| 会话数据 | 24 小时 | `session:{session_id}` |
| 研究取消标志 | 5 分钟 | `research:cancel:{session_id}` |

## 6.3 Git 规范化

### 6.3.1 如何制作干净的 Commit

**Commit 原则：**

1. **一个 commit 只做一件事**：不要在一个 commit 中同时修改后端 API 和前端样式
2. **先 commit 后 push**：本地多做几个小 commit，最后 push 之前可以 squash
3. **Commit 之前检查 diff**：`git diff --staged` 确认改动内容

**操作流程：**

```bash
# 1. 查看所有变更
git status

# 2. 按模块分阶段添加
git add backend/app/config/industry_config.py  # 只加相关文件
git add backend/app/service/deep_research_v2/ecommerce_mock_data_baby.py

# 3. 确认改动
git diff --staged

# 4. 提交
git commit -m "feat: add baby industry support"
```

**不要做：**

```bash
# 不好：一次把所有文件都加上
git add .
git commit -m "update"  # 模糊的消息
```

### 6.3.2 Commit Message 规范

采用 **Conventional Commits** 规范：

```
<type>(<scope>): <subject>

<body>
```

**Type 类型：**

| Type | 含义 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat: add baby industry mock data` |
| `fix` | 修复 Bug | `fix: resolve SSE disconnection on timeout` |
| `refactor` | 重构 | `refactor: extract chart generation to separate service` |
| `docs` | 文档 | `docs: add debugging guide chapter 5` |
| `style` | 代码风格 | `style: format with prettier` |
| `test` | 测试 | `test: add e2e test for food industry` |
| `chore` | 构建/工具 | `chore: update dependencies` |
| `perf` | 性能 | `perf: add React.memo to Chart component` |

**Scope 范围：**

| Scope | 含义 |
|-------|------|
| `backend` | 后端整体 |
| `frontend` | 前端整体 |
| `agent` | Agent 相关 |
| `config` | 配置相关 |
| `api` | API 接口 |
| `chart` | 图表相关 |
| `docker` | Docker 配置 |

**好的 Commit Message 示例：**

```
feat(agent): add DataQueryer agent for ecommerce data retrieval

Replaces DeepScout for ecommerce scenarios. Supports mock data mode
and AI extraction mode. Integrates with 4 industries (fashion, beauty,
digital, food).

Closes #42
```

```
fix(chart): prevent ECharts blank rendering on tab switch

Call chart.resize() when tab becomes visible. Previously the chart
would render in a zero-size container if initialized while hidden.
```

### 6.3.3 分支策略

推荐使用 **Git Flow 简化版**：

```
main
  ├── develop
  │     ├── feat/add-baby-industry
  │     ├── fix/sse-timeout
  │     └── feat/add-radar-chart
  └── release/v1.0
```

**分支命名规则：**

| 分支类型 | 命名格式 | 示例 |
|---------|---------|------|
| 主分支 | `main` | — |
| 开发分支 | `develop` | — |
| 功能分支 | `feat/<description>` | `feat/add-baby-industry` |
| 修复分支 | `fix/<description>` | `fix/sse-disconnect` |
| 发布分支 | `release/v<version>` | `release/v1.0` |

**操作示例：**

```bash
# 从 develop 创建功能分支
git checkout develop
git checkout -b feat/add-baby-industry

# 开发完成后合并回 develop
git checkout develop
git merge feat/add-baby-industry

# 发布时从 develop 到 main
git checkout main
git merge develop
git tag v1.0
```

### 6.6.1 当前架构分析

当前数据流程：

```
用户问题 -> DataQueryer Agent -> get_mock_data_for_industry(industry_id)
```

**2. 创建数据访问层（Repository 模式）**

在 `D:/industry_information_assistant/backend/app/service/` 中创建 `ecommerce_repository.py`：

```python
from sqlalchemy.orm import Session
from typing import List, Dict, Any
from backend.app.models.ecommerce import EcommerceProduct, EcommerceSale, EcommerceReview

class EcommerceRepository:
    def __init__(self, db: Session):
        self.db = db

    def get_products(self, industry_id: str) -> List[EcommerceProduct]:
        return self.db.query(EcommerceProduct).filter(
            EcommerceProduct.industry_id == industry_id
        ).all()

    def get_monthly_sales(self, product_ids: List[str], year: int, months: List[int]) -> List[EcommerceSale]:
        return self.db.query(EcommerceSale).filter(
            EcommerceSale.product_id.in_(product_ids),
            EcommerceSale.year == year,
            EcommerceSale.month.in_(months)
        ).all()

    def get_reviews(self, product_ids: List[str]) -> List[EcommerceReview]:
        return self.db.query(EcommerceReview).filter(
            EcommerceReview.product_id.in_(product_ids)
        ).all()

    def get_all_data(self, industry_id: str) -> Dict[str, Any]:
        """获取指定行业的所有数据（兼容现有的返回格式）"""
        products = self.get_products(industry_id)
        product_ids = [p.id for p in products]
        sales = self.get_monthly_sales(product_ids, 2024, [3, 4, 5, 6, 7, 8])
        reviews = self.get_reviews(product_ids)

        return {
            "products": [p.__dict__ for p in products],
            "monthly_sales": [s.__dict__ for s in sales],
            "reviews": [r.__dict__ for r in reviews],
            # ... 其他数据表
        }
```

```python
class DataQueryer(BaseAgent):
    def __init__(self, ..., use_db: bool = False):
        self.use_mock = use_mock  # 现有
        self.use_db = use_db       # 新增

    async def process(self, state: ResearchState) -> ResearchState:
        if self.use_db:
            return await self._query_from_database(state)
        elif self.use_mock:
            return await self._query_mock_data(state)
        else:
            return await self._query_via_ai_extraction(state)

    async def _query_from_database(self, state: ResearchState) -> ResearchState:
        """从数据库查询电商数据"""
        from backend.app.service.ecommerce_repository import EcommerceRepository
        from backend.app.core.database import SessionLocal

        db = SessionLocal()
        try:
            repo = EcommerceRepository(db)
            data = repo.get_all_data(state["industry_id"])
            # 转换为 ResearchState 需要的格式
            state["facts"] = self._convert_data_to_facts(data)
            state["data_points"] = self._extract_data_points(data)
        finally:
            db.close()
        return state
```

```python
"""

import sys

        for p in data["products"]:
```

## 常见坑

### 坑 1：添加新行业后忘记更新前端的 TypeScript 类型

**现象：** 后端能正常返回新行业数据，但前端 TypeScript 编译报错，或者行业选择器不显示新行业。

**原因：** `industry.ts` 中的 `INDUSTRY_CONFIGS` 是硬编码数组，没有和后端配置联动。

**解决方法：** 每次添加新行业，前后端配置文件都要同步更新。可以考虑从后端 API 动态获取行业列表替代前端硬编码：

```typescript
// 理想方案：从 API 获取
const fetchIndustries = async () => {
  const response = await api.getIndustries()
  // 动态更新 industryState
}
```

### 坑 2：Mock 数据文件的 lazy import 路径问题

**现象：** 创建了新的 mock 数据文件但系统加载不到。

**原因：** `get_mock_data_for_industry()` 中使用了多重 import fallback（如 `ecommerce_mock_data.py` 第 336-366 行），如果新的 lazy import 路径没有正确配置，就会 fallback 到默认数据。

**解决方法：** 确保 lazy import 中有 3 种回退路径：

```python
try:
    from .ecommerce_mock_data_baby import get_baby_mock_data
except ImportError:
    try:
        from app.service.deep_research_v2.ecommerce_mock_data_baby import get_baby_mock_data
    except ImportError:
        import ecommerce_mock_data_baby
        get_baby_mock_data = ecommerce_mock_data_baby.get_baby_mock_data
```

### 坑 3：React.memo 的比较函数写错导致数据不更新

**现象：** 使用 `React.memo` 包装组件后，图表数据变化但界面不更新。

**原因：** 自定义比较函数中使用了浅比较，深层嵌套的对象变化没有检测到。

**解决方法：** 对于复杂对象，使用 `JSON.stringify` 或专门的深比较库：

```typescript
React.memo(Component, (prev, next) => {
  return JSON.stringify(prev.config) === JSON.stringify(next.config)
})
```

或者干脆不写比较函数，让 React 使用默认的浅比较（适用于大部分场景）。

### 坑 4：雷达图 radar.indicator 的 max 值设置不合理

**现象：** 不同数据系列的值差异大，雷达图上所有数据点挤在一起或超出范围。

**解决方法：** 根据实际数据动态设置 max 值：

```typescript
const allValues = series_data.flatMap(s => s.data)
const maxValue = Math.max(...allValues) * 1.2  // 留 20% 余量

radar: {
  indicator: indicators.map(ind => ({
    name: ind.name,
    max: Math.ceil(maxValue / 10) * 10  // 向上取整到十位
  }))
}
```

### 坑 5：SQLAlchemy session 未正确关闭导致连接泄漏

**现象：** 运行一段时间后报 `QueuePool limit overflow` 错误。

**原因：** 在异步代码中使用同步的 SQLAlchemy session，忘记在 finally 块中关闭。

**解决方法：** 始终使用 try/finally 或 with 语句管理 session：

```python
db = SessionLocal()
try:
    # ... 数据库操作
    db.commit()
except Exception:
    db.rollback()
    raise
finally:
    db.close()  # 关键！
```

或使用 FastAPI 的依赖注入（自动管理生命周期）：

```python
def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```
