## 2.2 逐项安装

### 2.2.1 Python 3.10+

**Windows:**
1. 访问 [python.org](https://python.org/downloads/)
2. 下载 Python 3.10+ 安装包
3. **重要**：勾选 "Add Python to PATH"
4. 点击 Install，完成后验证：

```bash
$ python --version
Python 3.11.9  # 只要 ≥ 3.10 即可
```

> ⚠️ **注意**：如果命令行提示找不到 python，把 `C:\Users\你的用户名\AppData\Local\Programs\Python\Python311` 加到系统环境变量 PATH 中。

**macOS:**
```bash
$ brew install python@3.11
$ python3 --version
```

**Linux (Ubuntu):**
```bash
$ sudo apt update
$ sudo apt install python3.11 python3.11-venv python3.11-dev
$ python3.11 --version
```

## 2.4 启动基础设施（Docker）

项目根目录的 `docker-compose.yml` 定义了 6 个服务：

```bash
# 在项目根目录执行
$ docker compose up -d
```

> 🔧 **实战**：观察输出日志，确认 6 个服务依次启动。首次启动需要拉取镜像，可能需要几分钟。

验证所有服务都在运行：

```bash
$ docker compose ps
```

你应该看到：

```
NAME                STATUS    PORTS
ecom-postgres-1     Up        0.0.0.0:5432->5432/tcp
ecom-redis-1        Up        0.0.0.0:6379->6379/tcp
ecom-etcd-1         Up        2379-2380/tcp
ecom-minio-1        Up        9000-9001/tcp
ecom-milvus-1       Up        0.0.0.0:19530->19530/tcp
ecom-elasticsearch-1 Up        9200->9200/tcp, 9300->9300/tcp
```

### 各服务端口速查

| 服务 | 端口 | 访问方式 |
|------|------|----------|
| PostgreSQL | 5432 | `psql -h localhost -U postgres` |
| Redis | 6379 | `redis-cli` |
| Milvus | 19530 | Python SDK |
| Elasticsearch | 9200 | `curl http://localhost:9200` |
| MinIO Console | 9001 | 浏览器 http://localhost:9001 |

### 连接测试

测试 PostgreSQL：

```bash
$ docker exec -it $(docker ps -qf "name=postgres") psql -U postgres -c "SELECT 1"
 ?column?
----------
        1
```

测试 Redis：

```bash
$ docker exec -it $(docker ps -qf "name=redis") redis-cli PING
PONG
```

> ⚠️ **注意**：如果某个服务启动失败，查看日志：
> ```bash
> $ docker compose logs postgres  # 替换为失败的服务名
> ```

## 2.5 配置 API Key

### 2.5.1 申请阿里百炼 API Key

1. 打开 [bailian.console.aliyun.com](https://bailian.console.aliyun.com)
2. 注册/登录阿里云账号
3. 在"模型广场"找到 DeepSeek-V3.2 和 Qwen-Plus
4. 在"API-KEY 管理"创建新的 API Key
5. 复制保存（只显示一次！）

### 2.5.2 申请博查搜索 API Key（可选）

1. 打开 [open.bochaai.com](https://open.bochaai.com)
2. 注册账号，获取 API Key

### 2.5.3 配置到项目

在 `backend/` 目录下创建 `.env` 文件：

```bash
$ cd backend
$ cat > .env << 'EOF'
DASHSCOPE_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
BOCHA_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
EOF
```

> 💡 **提示**：`DASHSCOPE_API_KEY` 是必填的（LLM 调用需要），`BOCHA_API_KEY` 可选（不用网络搜索功能可以不加）。

## 常见坑

### 坑 1：Docker 启动后 PostgreSQL 端口冲突

**现象**：`Error starting userland proxy: listen tcp4 0.0.0.0:5432: bind: address already in use`

**原因**：本机已经安装了 PostgreSQL 占用了 5432 端口。

**解决**：停掉本机的 PostgreSQL 服务，或者修改 `docker-compose.yml` 中的端口映射：
```yaml
ports:
  - "5433:5432"  # 改用 5433
```

### 坑 2：pip install 报错

**现象**：`ERROR: Could not find a version that satisfies the requirement`

**原因**：Python 版本太低或太高。

**解决**：确认 Python 版本 ≥ 3.10 且 < 3.13。

### 坑 3：前端 npm install 报错

**现象**：`npm ERR! code ERESOLVE`

**原因**：依赖版本冲突。

**解决**：
```bash
$ rm -rf node_modules package-lock.json
$ npm install --legacy-peer-deps
```

### 坑 4：API Key 无效

**现象**：后端日志出现 `AuthenticationError` 或 `401 Unauthorized`

**原因**：API Key 配置不正确或已过期。

**解决**：
1. 确认 `backend/.env` 文件存在且 Key 正确
2. 确认 Key 有额度，在阿里百炼控制台查看
3. 重新申请 Key 试试
